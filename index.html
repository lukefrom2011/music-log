# Setup Guide: Album Ratings (Multi-User) This guide takes you from zero to a live, multi-user site in about 20 minutes. **What you're deploying:** - **Frontend:** Single `index.html` on GitHub Pages (free) - **Backend:** Supabase — handles auth, database, and row-level security (free tier) --- ## Part 1 — Supabase Setup ### 1.1 Create a project 1. Go to [supabase.com](https://supabase.com) and sign up for a free account 2. Click **New Project** 3. Give it a name (e.g. `album-ratings`), set a strong database password, choose a region close to you 4. Wait ~2 minutes for it to provision --- ### 1.2 Create the database tables Go to your project's **SQL Editor** (left sidebar) and run this entire block: ```sql -- User-added albums create table public.user_albums ( id bigserial primary key, user_id uuid references auth.users(id) on delete cascade not null, album text not null, artist text not null, release_date date, favorite_song text, score numeric(5,2) not null, art_url text, detail jsonb, created_at timestamptz default now() ); -- iTunes art cache (shared across all users — no PII) create table public.art_cache ( seed_id text primary key, art_url text not null, fetched_at timestamptz default now() ); -- Per-user manually uploaded art create table public.manual_art ( user_id uuid references auth.users(id) on delete cascade not null, album_id text not null, art_url text not null, updated_at timestamptz default now(), primary key (user_id, album_id) ); -- Per-user score overrides for seed albums create table public.score_overrides ( user_id uuid references auth.users(id) on delete cascade not null, seed_id text not null, override_data jsonb not null, updated_at timestamptz default now(), primary key (user_id, seed_id) ); -- ── Row Level Security ── -- Each table is locked down so users can only touch their own rows. alter table public.user_albums enable row level security; alter table public.art_cache enable row level security; alter table public.manual_art enable row level security; alter table public.score_overrides enable row level security; -- user_albums: full access to own rows create policy "users manage own albums" on public.user_albums for all using (auth.uid() = user_id) with check (auth.uid() = user_id); -- art_cache: any logged-in user can read; any logged-in user can insert/update -- (it's just iTunes URLs, no personal data) create policy "art_cache read" on public.art_cache for select to authenticated using (true); create policy "art_cache write" on public.art_cache for insert to authenticated with check (true); create policy "art_cache update" on public.art_cache for update to authenticated using (true); -- manual_art: full access to own rows create policy "users manage own manual art" on public.manual_art for all using (auth.uid() = user_id) with check (auth.uid() = user_id); -- score_overrides: full access to own rows create policy "users manage own overrides" on public.score_overrides for all using (auth.uid() = user_id) with check (auth.uid() = user_id); ``` Click **Run**. You should see "Success. No rows returned." --- ### 1.3 Configure Auth settings In Supabase, go to **Authentication → Providers**: 1. **Email** should already be enabled. Keep it on. 2. Under **Authentication → Email Templates**, the default "Confirm signup" and "Reset password" emails work out of the box — Supabase sends them automatically from their domain. Go to **Authentication → URL Configuration**: 1. Set **Site URL** to your GitHub Pages URL: `https://YOUR-USERNAME.github.io/album-ratings` 2. Under **Redirect URLs**, add: `https://YOUR-USERNAME.github.io/album-ratings` `https://YOUR-USERNAME.github.io/album-ratings/` --- ### 1.4 Get your API credentials Go to **Project Settings → API** (the gear icon in the sidebar): Copy these two values — you'll need them in the next step: - **Project URL** (looks like `https://gfzzemjcaqsuufjukkvv.supabase.co`) - **anon / public key** (a long JWT string — this one is safe to put in your HTML) sb_publishable_2fgxRvmKYZzii2jtuyRX2w_wbOZ9QbI > **Note:** Never use the `service_role` key in frontend code.
